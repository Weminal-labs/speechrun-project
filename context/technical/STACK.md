# Tech Stack

> Generated by setup wizard. Update as decisions evolve.

## Frontend
- **React** with **TypeScript** — component-based UI
- **Vite** — fast dev server and build tool
- **Tailwind CSS** — utility-first styling, dark theme

## Backend
- **Cloudflare Workers** — serverless edge runtime (TypeScript)
- **Durable Objects** — stateful session store with WebSocket support and built-in SQLite (plain `DurableObject` class, not agents SDK)
- **Workers AI** — LLM inference (Llama 3.1 8B FP8 in Phase 2, can upgrade to 70B)

## Database
- **Durable Object SQLite** — per-agent persistent storage for context, transcripts, and dialogue state
- **R2** — object storage for generated audio files

## Auth
- None — open tool, no user accounts required

## Infrastructure
- **Cloudflare Workers** — hosting, compute, and edge deployment
- **Cloudflare R2** — audio file storage (S3-compatible)
- **Cloudflare AI Gateway** (optional) — model routing and observability

## Key Third-Party Services
- **ElevenLabs** — text-to-speech and Text to Dialogue API for podcast audio generation
- **GitHub REST API** — codebase fetching (repo tree + file contents)
- **Workers AI** (Cloudflare) — LLM inference for context generation and agent dialogue (e.g. Llama 3.1 70B)

## Rationale
- **Cloudflare-native stack** — hackathon is Cloudflare x ElevenLabs, so the entire backend runs on Cloudflare's platform. Durable Objects give us stateful agents with WebSocket push and built-in SQLite, eliminating the need for a separate database.
- **React + Vite** — lightweight, fast iteration, included in the official starter kit.
- **Tailwind** — rapid styling without leaving markup, dark theme support out of the box.
- **No auth** — this is a hackathon demo tool. Users paste a GitHub URL and get a podcast. No accounts, no friction.
- **R2 for audio** — podcast audio files can be large; R2 gives us cheap, fast object storage on the same platform.
